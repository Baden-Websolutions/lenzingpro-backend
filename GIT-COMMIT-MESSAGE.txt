feat: Add JWT-Bearer token exchange authentication flow

Implements a parallel JWT-based authentication flow based on RFC 7523
and the SAP AppGyver auth-flow pattern. This flow runs alongside the
existing CDC-OIDC flow without modifying any existing files.

## Features

- JWT-Bearer token exchange (RFC 7523)
- JWKS-based JWT validation
- Token caching for performance (NodeCache)
- Session management compatible with existing SessionStore
- Complete API endpoints for login, session, refresh, logout
- Protected route examples
- Comprehensive documentation and tests

## Integration

Add one line to src/server.ts after registerUserProtectedRoutes:

```typescript
import { registerJWTAuthentication } from "./server-jwt-integration.js";
await registerJWTAuthentication(app, env, sessionStore);
```

## Dependencies

- Requires: node-cache (new)
- Existing: jose, fastify, @fastify/cookie

## API Endpoints

Authentication:
- POST /jwt-auth/login
- GET /jwt-auth/session
- POST /jwt-auth/refresh
- POST /jwt-auth/logout

Protected (examples):
- GET /jwt-protected/profile
- GET /jwt-protected/commerce-token
- POST /jwt-protected/test-commerce

## Security

- JWKS signature verification
- Issuer and audience validation
- HttpOnly, Secure, SameSite cookies
- Token caching with SHA256 keys
- Session prefix separation (jwt_*)
